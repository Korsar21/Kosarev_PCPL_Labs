 #  Telegram RPG Dungeon Crawler

Текстовая RPG-игра для Telegram, где игрок исследует процедурно генерируемое подземелье, сражается с монстрами, собирает оружие и зелья, проходит квесты на внимательность и ищет заветный ключ победы.

##  Описание

Проект представляет собой полноценную текстовую RPG-игру, реализованную как Telegram-бот на Python. Игрок перемещается по случайно генерируемым комнатам подземелья, принимая решения с помощью кнопок интерфейса.

##  Реализованный функционал

###  Система подземелья
- **Процедурная генерация** — каждая комната создаётся случайным образом при входе
- **Древовидная структура** — комнаты связаны между собой (вперёд, влево, вправо)
- **Навигация** — возможность возвращаться в уже посещённые комнаты
- **Случайное содержимое** — монстры, оружие, зелья и ключ появляются с разной вероятностью

###  Боевая система
- **Пошаговые сражения** с монстрами
- **30 уникальных монстров** с разными характеристиками (здоровье, урон, уровень)
- **Система уровней** — сложность монстров зависит от глубины прохождения
- **Блокировка перемещения** — нельзя пройти дальше, пока монстр жив

###  Система оружия
- **19 видов оружия** — от ножа до плазменной пушки
- **Прогрессия урона** — мощное оружие появляется на более глубоких уровнях
- **Обмен оружия** — можно менять текущее оружие на найденное в комнате

###  Система зелий
- **Случайная сила зелья** — восстанавливает от 5 до 12 HP
- **Накопление** — можно носить одно зелье
- **Объединение зелий** — при взятии нового зелья эффекты суммируются с бонусом +10%
- **Стратегический выбор** — когда выпить зелье решает игрок

###  Система квестов
- **Испытание на память** — при входе в некоторые комнаты
- **Механика:** быстрый показ 3 символов → игрок вводит их по памяти
- **6 различных символов** (круги, треугольники, квадраты разных цветов)
- **Последствия:** успех = вход в комнату, провал = возврат назад
- **Повышенный шанс ключа** в комнатах с квестом (20% против 5%)

###  Условия победы и поражения
- **Победа** — найти и взять ключ
- **Поражение** — здоровье персонажа опускается до 0
- **Защита ключа** — ключ может охранять монстр или квест

###  Система игрока
- **Кастомизация** — выбор имени персонажа
- **Инвентарь** — текущее оружие и зелье
- **Здоровье** — начальное значение 50 HP

###  Интерфейс
- **Кнопочное управление** — все действия через Reply-клавиатуру
- **Информативные сообщения** — статус игрока, содержимое комнаты
- **Эмодзи-оформление** — наглядное отображение состояния игры

##  Технологии

- **Python 3**
- **pyTelegramBotAPI (telebot)** — взаимодействие с Telegram Bot API
- **Модульная архитектура** — разделение на классы и модули
